# 🚀 팀 프로젝트 협업 가이드 (FE & AI)

이 문서는 백엔드(BE) 서버를 중심으로 프론트엔드(FE)와 AI 모듈이 데이터를 어떻게 주고받아야 하는지 정리한 최종 가이드입니다.

---

## 1. 서버 접속 및 인증 정보

### 🔗 접속 주소
*   **Base URL:** 서버 실행 시 공유되는 Cloudflare 랜덤 URL (예: `https://...trycloudflare.com`)
*   **API 문서:** `{Base URL}/docs` (Swagger) 또는 `{Base URL}/redoc`
    *   **문서 열람용 ID:** `admin`
    *   **문서 열람용 PW:** `quiz-guide-secret`

### 🔑 인증 방식 (JWT)
1.  **회원가입:** `POST /register` 를 통해 각자 계정 생성.
2.  **로그인:** `POST /login` 에 이메일/비번 전송 시 `access_token` 발급.
3.  **API 호출:** 모든 요청 헤더에 아래 내용을 포함해야 함.
    *   `Authorization: Bearer <발급받은_토큰>`

## 2. 파트별 데이터 역할 분담 (Read/Write)

| 파트 | 대상 테이블 | 읽기 (📖 Read) | 쓰기 (✍️ Write) | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **FE** | `Lectures` | 목록 및 본문 조회 | 강의 자료 업로드 | 사용자가 올린 데이터 |
| | `AITasks` | **진행률 및 상태** | - | 실시간 로딩바 구현용 |
| | `Concepts` | 추출된 개념 목록 | - | AI의 결과물 출력 |
| | `QuizQuestions`| 퀴즈 문항 정보 | - | 문제 풀기 화면 구성 |
| | `QuizResults` | 최종 점수 및 피드백 | **점수 및 선택 답안** | 사용자 풀이 결과 제출 |
| **AI** | `Lectures` | 분석할 원본 텍스트 | - | - |
| | `AITasks` | - | **상태 및 진행률** | 작업 단계별 업데이트 필수 |
| | `Concepts` | - | **핵심 개념 데이터** | 분석 완료 후 DB 삽입 |
| | `QuizQuestions`| - | **퀴즈 문항 데이터** | 문제 생성 후 DB 삽입 |

## 3. 개발 시 필수 주의사항

### 🔄 비동기 작업 흐름 (Flow)
1.  **FE:** 강의 업로드 또는 퀴즈 생성 요청 ➔ 서버로부터 `task_id` 수신.
2.  **AI:** 외부에서 작업을 감지하거나 요청 수신 ➔ `AITasks` 테이블의 상태를 `PROCESSING`으로 변경하며 일 진행.
3.  **FE (폴링):** `GET /tasks/{task_id}`를 주기적(예: 2~3초 간격)으로 호출하여 상태 확인.
4.  **AI:** 작업 완료 시 `Concepts` 또는 `QuizQuestions`에 결과 삽입 후, `AITasks` 상태를 `COMPLETED`로 최종 변경.
5.  **FE:** 상태가 `COMPLETED`로 확인되면 결과를 화면에 렌더링.

### ⚠️ 데이터 정합성 주의
*   **UUID 매칭:** AI가 데이터를 DB에 넣을 때, 반드시 백엔드에서 생성된 `lecture_id`나 `quiz_id`와 정확히 매칭시켜야 합니다.
*   **상황 업데이트:** AI가 작업을 시작할 때 `status`를 바꾸지 않으면, FE에서는 서버가 죽은 것으로 오해할 수 있습니다. 상태 업데이트를 꼼꼼히 해주세요.
*   **CORS:** 현재 백엔드는 모든 도메인(`*`)에서의 접속을 허용하고 있으므로 로컬 개발 환경에서 주소만 맞추면 바로 통신 가능합니다.
